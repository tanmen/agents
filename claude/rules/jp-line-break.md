# 日本語 UI テキストの改行（不自然な折り返しを避ける）

日本語の UI テキストは、文節の途中で折り返して意味のまとまりを割らない。ブラウザ既定の禁則処理は「ほぼどこでも改行可」なので、放置すると「…それ / が3軸採点。」のように語のまとまりが割れたり、最終行に数文字だけ残る孤立行が出て読みにくくなる。frontend / component を **生成・修正するたびに**、次を既定で適用する（design 系 skill＝frontend-design / readability-review / flow-layout が走るときも同じ観点で見る）。

- **文節区切りで折り返す**: 見出し・リード文・短い説明文には `word-break: auto-phrase` を付け、文節（bunsetsu）境界で改行させる。「それ｜が」のような不自然な分割を防ぐ最も効く一手。
- **孤立行・尾切れを防ぐ**: 本文は `text-wrap: pretty`、見出しや 2〜3 行のリード文は `text-wrap: balance` で行長を均し、最終行に 1〜2 文字だけ残す状態を避ける。
- **フォールバック / 明示制御**: `word-break: auto-phrase` 未対応ブラウザ（Safari / Firefox 等）も対象なら **BudouX**（文節境界に `<wbr>` を挿入）を併用。折り返してほしくない短いラベルや固有名詞は `white-space: nowrap`、途中で切りたくない語のまとまりは `nowrap` を当てた `<span>` で囲む。
- **measure（1 行の字数）を狭くしすぎない**: 横に十分な余白があるのに極端に狭い幅で折り返しているとき（例＝右に大量の空きがあるのに ~20 字強で改行）、原因は文字組みではなく container の `max-width` / grid 列幅であることが多い。**まず幅側を疑って直す**。日本語本文の可読な measure は概ね全角 35〜45 字が上限の目安で、それを大きく下回る狭さは避ける。
